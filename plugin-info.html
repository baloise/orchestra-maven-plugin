Plugin Documentation

Goals available for this plugin:

Goal Description
orchestra:help Display help information on orchestra-maven-plugin.
Call mvn orchestra:help -Ddetail=true -Dgoal=<goal-name> to display parameter details.
orchestra:scenario-deploy Deploys ( or redeploys ) a scenario ( PSC file ).

Example usage: 
mvn com.baloise.maven:orchestra-maven-plugin scenario-deploy -Duser=admin -Dpassword=*** -Dserver=https://orchestra.example.com:8019 -DpscFile=/tmp/my.psc -landscapeFile=/tmp/landscape.json
If you use a pom.xml feel free to omit pscFile
orchestra:scenario-package
Maven layout
Put your PSC source files under /src/main/orchestra and run 
mvn com.baloise.maven:orchestra-maven-plugin scenario-package
to package PSC file into the target folder.

or include the plugin in your pom.xml and run 
mvn package

Package PSC source folder without Maven project layout / POM
mvn -DpscFile=/path/to/my.psc -DsourceFolder=/path/to/myPSCsource -Dexclude=DEFAULT com.baloise.maven:orchestra-maven-plugin scenario-package
Exclude parameter is recommended so you don't accidentally package .git folder or similar.

System Requirements

The following specifies the minimum requirements to run this Maven plugin:

Maven 3.3.9
JDK 1.8
Memory No minimum requirement.
Disk Space No minimum requirement.

Usage

You should specify the version in your project's plugin configuration:

<project>
  ...
  <build>
    <!-- To define the plugin version in your parent POM -->
    <pluginManagement>
      <plugins>
        <plugin>
          <groupId>com.baloise.maven</groupId>
          <artifactId>orchestra-maven-plugin</artifactId>
          <version>0.5.3</version>
        </plugin>
        ...
      </plugins>
    </pluginManagement>
    <!-- To use the plugin goals in your POM or parent POM -->
    <plugins>
      <plugin>
        <groupId>com.baloise.maven</groupId>
        <artifactId>orchestra-maven-plugin</artifactId>
        <version>0.5.3</version>
      </plugin>
      ...
    </plugins>
  </build>
  ...
</project>

For more information, see "Guide to Configuring Plug-ins"
